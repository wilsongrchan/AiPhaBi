# 愛發筆輸入法 · AiPhaBi

一套自製的形碼中文輸入法，以及用來設計它的工具。

字根表不是手工填出來的，而是**從逐字取碼學回來的**：你在某個字上圈幾筆、按一個字母，
那個形狀就成為該字母底下的一個字根；同形自動合併計次，新形則新增一條。
字根累積起來之後，反過來替沒取過碼的字**預測整字拆解**。兩邊共用同一份資料，改一邊、另一邊即時跟著變。

## 三個頁面

| 頁面 | 用途 |
| --- | --- |
| `/` 字根表 | 26 個字母下的字根，按「取形意圖」分組，可拖放改組別／等級 |
| `/annotate` 逐字取碼 | 點筆畫 → 按字母 → 成一個字根；看預測拆解；查三地官方筆順 |
| `/rules` 取碼原則 | 筆順、能合不分、孤筆略過、碼長上限等；標「會執行」的會實際影響預測 |

## 跑起來

```bash
python3 fetch_data.py     # 第一次：下載字形資料（約 30 MB）
python3 server.py         # → http://localhost:8777
```

## 字根怎麼表示

字根不受 Unicode 限制 —— 很多字根（例如「笑」的竹頭左半、「制」的首三筆）根本沒有碼位。
所以一個字根是**某個字的一組筆畫**：

```json
{ "glyph": { "src": "笑", "strokes": [0, 1, 2] }, "count": 3, "seen": ["笑", "第", "筆"] }
```

比對字根時用**筆畫中線**，不用點陣圖：部件在不同字裡會被壓扁拉長，點陣圖比對分不開
（實測同一個「月」距離 0.54，比某些不同字根還遠）；中線比對實測同字根 0.01–0.17、不同字根 0.49 以上，
所以預設合併門檻 0.25 落在中間。

## 資料來源

程式碼在 git 裡，字形資料不在（各有授權，`fetch_data.py` 會自行下載）：

- **大陸筆順** — [makemeahanzi](https://github.com/skishore/makemeahanzi)（Arphic 字型授權），9,574 字的逐筆輪廓與中線。取碼即以此為準。
- **台灣筆順** — 教育部《國字標準字體筆順學習網》資料，經 [g0v/zh-stroke-data](https://github.com/g0v/zh-stroke-data)，4,847 常用字。
- **香港筆順** — 教育局《香港小學學習字詞表》的筆順示範（Adobe Animate/CreateJS 匯出的向量資料，由 `hk.py` 解碼），隨用隨抓並快取。**屬教育局版權，只作個人參考，不隨本專案散布。**
- **字頻** — 由 [rime-essay](https://github.com/rime/rime-essay) 統計，決定取碼佇列的先後。
- **倉頡對照** — 碼表取自 [rime-cangjie](https://github.com/rime/rime-cangjie)（GPL）。
  「哪一筆屬於哪一碼」則由 `cangjie_map.py` 從[倉頡字典](https://倉頡字典.com)的拆碼圖推出來
  （沿筆畫中線取色 → 同色歸組 → 對到碼），隨用隨抓並快取。推出來的是事實資料，
  用我們自己的筆畫重畫；圖片不保留、不散布。**屬該站版權，只作個人對照。**

## 你的資料

`data/zigen.json`（字根表）、`data/codes.json`（碼表）、`data/rules.json`（取碼原則）會進 git。
每次存檔前自動備份到 `data/backups/`（不入 git，保留最近 200 份）。

跑測試時請用另一個資料目錄，免得動到真資料：

```bash
AIPHABI_DATA=/tmp/aiphabi-test python3 server.py
```
